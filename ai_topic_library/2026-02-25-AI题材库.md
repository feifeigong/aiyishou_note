# 今日 AI 资讯（2026-02-25）

## 1) Anthropic 发布 RSP 3.0（2月24日）
Anthropic 更新了 Responsible Scaling Policy 到 3.0 版。核心变化是：把“公司可单独执行的措施”和“行业层面的建议”分开，并引入更高透明度机制（Frontier Safety Roadmap、定期 Risk Reports、特定情形下外部评审）。

来源：<https://www.anthropic.com/news/responsible-scaling-policy-v3>

---

## 2) Anthropic 披露大规模“蒸馏攻击”案例（2月23日）
Anthropic 表示识别到针对 Claude 能力提取的工业化活动，涉及大量账号与调用流量，并公布了其检测、防护与跨机构协同思路。该议题指向模型安全、知识产权与地缘竞争的交叉风险。

来源：<https://www.anthropic.com/news/detecting-and-preventing-distillation-attacks>

---

## 3) OpenAI：SWE-bench Verified 不再适合作为前沿编码能力指标（2月23日）
OpenAI 在研究文章中指出 SWE-bench Verified 出现“测试缺陷 + 训练污染”问题，建议改用 SWE-bench Pro，且将停止报告 SWE-bench Verified 分数。对模型评测口径和横向对比会有直接影响。

来源：<https://openai.com/index/why-we-no-longer-evaluate-swe-bench-verified/>

---

## 4) Anthropic 推出 Claude Code Security 研究预览（2月20日）
Anthropic 发布面向防守方的 Claude Code Security（限量研究预览），主打代码漏洞识别与修复建议，并强调“人类审核后再执行”的流程。方向上是把前沿模型能力用于安全加固，而非仅效率提升。

来源：<https://www.anthropic.com/news/claude-code-security>

---

## 5) DeepMind：Gemini Deep Think 进入科研问题求解场景（2月11日）
Google DeepMind 介绍 Gemini Deep Think 在数学、物理、计算机科学研究中的进展，强调其在专业科研问题上的推理与迭代验证流程，并配套公开论文。

来源：<https://deepmind.google/blog/accelerating-mathematical-and-scientific-discovery-with-gemini-deep-think/>

---

## 6) OpenAI 已执行 ChatGPT 侧旧模型退场（2月13日生效，1月29日公告）
OpenAI 1月29日公告、2月13日执行：ChatGPT 侧退役 GPT-4o / GPT-4.1 / GPT-4.1 mini / o4-mini（API 侧当时说明“无变化”）。这意味着产品端默认体验进一步向新模型代际集中。

来源：<https://openai.com/index/retiring-gpt-4o-and-older-models/>

## 今日观察
1. “安全治理框架”从口号走向机制化披露：RSP 3.0、风险报告与外部评审正在成为行业共识方向。
2. 评测体系进入“反污染”阶段：基准分数不再只比高低，更要比可解释性与可复现性。
3. 安全能力产品化提速：AI 正从“生成内容”扩展到“漏洞发现/修复”等高价值防守场景。
